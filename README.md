# Веб‑приложение «Заявки в ремонтную службу»

Веб‑сервис для приёма и обработки заявок в ремонтную службу с ролями «диспетчер» и «мастер». Реализована защита от race condition при параллельных запросах.
## Запись 1
**Дата и время:** 2026-02-20 14:30

**Промпт:**
> Предложи структуру REST API для веб‑сервиса «Заявки в ремонтную службу»
## Технологии

* **Backend**: Node.js + Express
* **Frontend**: Vanilla JS + HTML/CSS
* **База данных**: SQLite (через Sequelize)
* **Контейнеризация**: Docker Compose
* **Тестирование**: Jest + Supertest

## Функционал

### Роли
* **Диспетчер** — создаёт, фильтрует, назначает мастеров и отменяет заявки.
* **Мастер** — берёт заявки в работу и завершает их.

### Заявки (поля)
* `clientName` (обязательно)
* `phone` (обязательно)
* `address` (обязательно)
* `problemText` (обязательно)
* `status`: `new | assigned | in_progress | done | canceled`
* `assignedTo` (мастер, может быть пустым)
* `createdAt`, `updatedAt`

### Страницы
1. **Создание заявки** — форма для подачи новой заявки (статус по умолчанию — `new`).
2. **Панель диспетчера**:
   * список всех заявок;
   * фильтр по статусу;
   * назначение мастера (статус `assigned`);
   * отмена заявки (статус `canceled`).
3. **Панель мастера**:
   * список заявок, назначенных на текущего мастера;
   * действие «Взять в работу» (`assigned → in_progress`);
   * действие «Завершить» (`in_progress → done`).
